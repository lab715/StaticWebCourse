Web Application Development #1: Intro

Internet

  • Computers send data via Internet
  • Peer to Peer ( IP and port )
  • Open connection, send data ( packages ), close connection. The data flow called socket.
  • Very slow at the early stage, so there are some "歷史包袱"

Web

  • Client-server model appeared
  • What to send? What to receive? Here came a standard: HTTP
  • Packages send from client to server called request
  • Packages send from server to client called response
  • Request and response are based on socket
  • People built browsers on clients to make requests and receive responses
